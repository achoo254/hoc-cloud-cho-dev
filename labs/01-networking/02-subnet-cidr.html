IPv4 Subnet & CIDR

Chia /24 thành 4 subnet /26 bằng tay, hiểu bit nào là network, bit nào là host, và vì sao broadcast khác với network.

Subnet không phải kiến thức "thi trắc nghiệm". Sai subnet = 2 máy cùng LAN không ping được nhau, hoặc route loop. Dev hiểu subnet = tự config Docker network, k8s CIDR, VPC AWS không bị đụng IP.

TL;DR Các khái niệm cốt lõi

Nhìn lại bảng này khi quên. Cột Why là phần cần nhớ nhất.

Core Bit Network vs Bit Host

CIDR /24 = 24 bit đầu là network, 8 bit cuối là host. Subnet mask là cái "mặt nạ" AND với IP để trích ra network address.

IP : 192.168.1.100 = 11000000.10101000.00000001.01100100 Mask/24 : 255.255.255.0 = 11111111.11111111.11111111.00000000 Network : AND = 11000000.10101000.00000001.00000000 → 192.168.1.0 # Broadcast = Network OR (NOT Mask): toàn bit 1 ở phần host Broad. : 192.168.1.255 = 11000000.10101000.00000001.11111111
Why cần tách network/host: Router chỉ quan tâm network part để forward. 2 máy cùng network = gửi frame trực tiếp (ARP + Ethernet). Khác network = phải qua gateway. Sai CIDR → máy "tưởng" chung mạng nhưng thực ra khác → gửi ARP rơi vào hư vô.
Why broadcast = toàn bit host bằng 1: Quy ước. Packet gửi tới broadcast address sẽ đến mọi host trong subnet đó (ARP, DHCP Discover). Đó là lý do usable hosts = 2^host − 2 (trừ network + broadcast).
