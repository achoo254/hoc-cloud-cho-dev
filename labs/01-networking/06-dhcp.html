DHCP — DORA flow

Câu hỏi cốt lõi: "VM vừa boot, chưa có IP — làm sao nó tự lấy được địa chỉ mà không cần config tay?"

Không phải học để thuộc tên DORA. Mục đích: debug được "máy không lên mạng sau khi boot", phân biệt DHCP fail vs static IP sai, hiểu tại sao broadcast, tại sao cần relay khi DHCP server ở subnet khác.

Core Tại sao DHCP dùng broadcast?

Đây là câu hỏi quan trọng nhất — khi hiểu được, toàn bộ DORA flow tự nhiên.

Client chưa có IP → không thể unicast. Giao thức TCP/IP yêu cầu source IP để gửi packet. Client vừa boot không có IP, chỉ biết địa chỉ broadcast toàn subnet 255.255.255.255. Nó dùng src=0.0.0.0 và dst=255.255.255.255 để "kêu to trong LAN" — mọi host nhận được, nhưng chỉ DHCP server mới phản hồi.
# Wireshark filter để bắt toàn bộ DORA
udp.port == 67 || udp.port == 68
# Port convention
Port 67 → DHCP server lắng nghe (receives Discover + Request)
Port 68 → DHCP client lắng nghe (receives Offer + Ack)

Lease time — IP không phải vĩnh viễn

Server "cho mượn" IP trong một khoảng thời gian (lease time). Client phải gia hạn trước khi hết hạn:

  • T1 (50% lease time): Client unicast thẳng tới DHCP server gốc để xin gia hạn
  • T2 (87.5% lease time): Nếu T1 thất bại, client broadcast rebind — chấp nhận bất kỳ server nào
  • 100%: IP hết hạn, client phải bắt đầu DORA lại từ đầu

DHCP Relay — vượt subnet

Broadcast không qua được router. Khi DHCP server ở subnet khác, relay agent (thường là router) nhận broadcast từ client, chuyển thành unicast tới server, rồi forward response ngược lại.

Client (broadcast) → [Relay Agent / Router] → DHCP Server (unicast)
DHCP Server → [Relay Agent] → Client (unicast/broadcast)

So sánh: DHCP vs Static IP

🔄 DHCP (Dynamic)

  • Tự động: không cần config tay từng máy
  • Scale: 1000 VM boot → 1000 IP, không cần admin can thiệp
  • Reclaim: IP được thu hồi khi máy tắt
  • Rủi ro: IP thay đổi mỗi lần renew → không làm DNS A record tay được
  • Dùng khi: workstation, VM dev, container, user device

📌 Static IP

  • Cố định: IP không thay đổi — DNS, firewall rule ổn định
  • Không phụ thuộc: DHCP server chết cũng không ảnh hưởng
  • Tốn công: config tay từng máy, dễ nhầm/conflict
  • Dùng khi: server, database, gateway, NAS, printer
